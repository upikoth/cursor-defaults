# push-all

Выполни **последовательно** для каждого Git-репозитория в корне workspace.

## 1. Список репозиториев

Из корня workspace выполни:

```bash
find . -maxdepth 2 -name .git -type d | sed 's|/.git||' | sort
```

Используй **только** эти каталоги (не заходи глубже и не ищи `.git` в других местах).

## 2. В каждом репозитории

1. `git status -sb` и при необходимости `git diff --stat` / `git diff --cached --stat`, чтобы понять суть изменений.
2. **Если** есть незакоммиченные изменения (включая неотслеживаемые файлы): `git add -A`, затем `git commit -m "..."` — сообщение **на русском**, по смыслу диффа, одно предложение.
3. **Если** рабочее дерево чистое — коммит не создавать.
4. `git push` (ветка по умолчанию, обычно `main`). Если нечего отправлять — ожидаемо сообщение вроде `Everything up-to-date`.

Порядок обхода — **как в отсортированном списке** из шага 1 (детерминированно).

## 3. Права и сбои

- Для `git commit` и `git push` запрашивай **git_write** и **network**; если коммит в `.cursor/` падает с ошибкой доступа к `index.lock` — повтори операцию с правами **all** для этого репозитория.
- Не создавай пустые коммиты (`--allow-empty` не использовать без явного запроса пользователя).

## 4. Отчёт пользователю

Краткая таблица: репозиторий → коммит создан (да/нет, хеш) → результат `push`.
